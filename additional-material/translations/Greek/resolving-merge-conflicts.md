# Τι είναι ένα merge conflict;

Όταν προσπαθείτε να συγχωνεύσετε ένα άλλο κλαδί (branch) στο τρέχον κλαδί εργασίας σας, παίρνετε αλλαγές από ένα διαφορετικό περιβάλλον και τις συνδυάζετε με τα τρέχοντα αρχεία εργασίας σας. Αν δύο άνθρωποι έχουν αλλάξει τις ίδιες γραμμές στο ίδιο αρχείο, ή αν ένας άνθρωπος αποφάσισε να το διαγράψει ενώ ο άλλος αποφάσισε να το τροποποιήσει, τότε το Git δεν μπορεί να προσδιορίσει ποια εκδοχή είναι η σωστή. Το Git θα σημειώσει το αρχείο ως με σύγκρουση (conflict), και θα πρέπει να την επιλύσετε πριν συνεχίσετε την εργασία σας.

# Πώς να επιλύσετε ένα merge conflict;

Όταν αντιμετωπίζετε ένα merge conflict, το Git θα σημειώσει το προβληματικό τμήμα του αρχείου, περικλείοντάς το με "<<<<<<<< HEAD" και ">>>>>>>>>>[όνομα άλλου κλαδιού]".

Το περιεχόμενο μετά το πρώτο σημάδι προέρχεται από το τρέχον κλαδί εργασίας σας. Μετά τις γωνιακές αγκύλες, το Git μας λέει από ποιο κλαδί προέρχονται οι αλλαγές. Μια γραμμή με "=======" χωρίζει τις δύο συγκρουόμενες αλλαγές.

Η δουλειά μας είναι τώρα να καθαρίσουμε αυτές τις γραμμές: όταν τελειώσουμε, το αρχείο θα πρέπει να φαίνεται ακριβώς όπως το θέλουμε. Συνιστάται να συμβουλευτείτε τον συνάδελφό σας που έγραψε τις αντικρουόμενες αλλαγές για να αποφασίσετε ποια έκδοση θα είναι τελική. Μπορεί να είναι μία από τις δικές σας εκδοχές, ή ίσως ένας συνδυασμός ανάμεσα στις δ

ύο.

π.χ.:
```
 <<<<<<< HEAD:mergetest
 Αυτή είναι η τρίτη μου γραμμή
 =======
 Αυτή είναι μία τέταρτη γραμμή που προσθέτω
 >>>>>>> 4e2b407f501b68f8588aa645acafffa0224b9b78:mergetest
```

`<<<<<<<`: Υποδεικνύει την αρχή των γραμμών που είχαν σύγκρουση. Οι πρώτες γραμμές είναι από το αρχείο που προσπαθούσατε να συγχωνεύσετε τις αλλαγές.
`=======`: Υποδεικνύει το σημείο χωρισμού που χρησιμοποιήθηκε για τη σύγκριση. Χωρίζει τις αλλαγές που ο χρήστης έχει δεσμεύσει (πάνω) από τις αλλαγές που προέρχονται από τη συγχώνευση (κάτω), ώστε να δούμε οπτικά τις διαφορές.
`>>>>>>>`: Υποδεικνύει το τέλος των γραμμών που είχαν σύγκρουση.

Λύνετε μια σύγκρουση επεξεργαζόμενοι το αρχείο και συγχωνεύοντας χειροκίνητα τα τμήματα του αρχείου που το Git δυσκολεύτηκε να συγχωνεύσει. Αυτό μπορεί να σημαίνει την απόρριψη των δικών σας αλλαγών ή των αλλαγών κάποιου άλλου, ή την προώθηση ενός συνδυασμού των δύο. Θα πρέπει επίσης να διαγράψετε τις γραμμές '<<<<<<<', '=======', και '>>>>>>>' από το αρχείο.

Αφού έχετε επιλύσει τη σύγκρουση, κάντε `git add`. Μην ξεχάσετε να τρέξετε τα τεστ, καθώς πρέπει να βεβαιωθείτε ότι έχετε επιλύσει τη σύγκρουση.

Μπορείτε επίσης να κατεβάσετε διάφορα πρόσθετα ανάλογα με το περιβάλλον ανάπτυξης (IDE) που χρησιμοποιείτε για μια ευκολότερη λύση των συγκρούσεων συγχώνευσης (merge conflicts). 

# Πώς να αναιρέσετε μια συγχώνευση;

Εάν θέλετε να αναιρέσετε μια συγχώνευση, τότε μπορείτε να χρησιμοποιήσετε την εντολή `git merge --abort`.
