# Αφαίρεση ενός αρχείου από το Git

Μερικές φορές, μπορεί να θέλετε να αφαιρέσετε ένα αρχείο από το Git αλλά όχι να το διαγράψετε από τον υπολογιστή σας. Μπορείτε να το κάνετε αυτό χρησιμοποιώντας την ακόλουθη εντολή:

``git rm <αρχείο> --cached``

## Τι συνέβη;

Το Git δεν θα παρακολουθεί πλέον τις αλλαγές στο αφαιρεθέν αρχείο. Όσον αφορά το Git, είναι σαν να είχατε διαγράψει το αρχείο. Αν ψάξετε για το αρχείο στο σύστημα αρχείων σας, θα παρατηρήσετε ότι εξακολουθεί να υπάρχει εκεί.

Παρατηρήστε ότι στο παράδειγμα παραπάνω, χρησιμοποιείται η σημαία `--cached`. Αν δεν προσθέσουμε αυτήν τη σημαία, το Git θα αφαιρέσει το αρχείο όχι μόνο από το αποθετήριο, αλλά και από το σύστημα αρχείων σας.

Αν κάνετε δέσμευση της αλλαγής με `git commit -m "Αφαίρεση αρχείου1.js"` και την ανεβάσετε στο απομακρυσμένο αποθετήριο χρησιμοποιώντας `git push origin master`, το απομακρυσμένο αποθετήριο θα αφαιρέσει το αρχείο.

## Πρόσθετες δυνατότητες

- Εάν θέλετε να αφαιρέσετε περισσότερα από ένα αρχεία, μπορείτε να τα συμπεριλάβετε όλα στην ίδια εντολή:

  `git rm αρχείο1.js αρχείο2.js αρχείο3.js --cached`

- Μπορείτε να χρησιμοποιήσετε ένα χαρακτήρα αντικατάστασης (*) για να αφαιρέσετε παρόμοια αρχεία. Για παράδειγμα, αν θέλετε να αφαιρέσετε όλα τα αρχεία .txt από το τοπικό σας αποθετήριο:

  `git rm *.txt --cached`
